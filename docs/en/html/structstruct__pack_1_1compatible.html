Detailed Description

template<typename T, uint64_t version>
struct struct_pack::compatible< T, version >

similar to std::optional<T>. However, its semantics are forward compatible field.

For example,

struct person_v1 {
int age;
std::string name;
};
struct person_v2 {
int age;
std::string name;
struct_pack::compatible<bool> id;
};

Note that extension fields can only be added at the end! Otherwise, compile-time error reported.

For example,

struct person2 {
int age;
struct_pack::compatible<int32_t> id;
std::string name;
struct_pack::compatible<bool> maybe;
};
person2 p = {.age=1,.id=1,.name="Hello",.maybe=false};
auto data = struct_pack::serialize(p);

compile error

error: static_assert failed, "The position of compatible<T> in struct is not
allowed!"

The value of compatible can be empty.
